# 3.1.1 失效数据

NoVisibility展⽰了在缺乏同步的程序中可能产⽣错误结果的⼀种情况：失效数据。当读线程查看ready变量时，可能会得到⼀个已经失效的值。除⾮在每次访问变量时都使⽤同步，否则很可能获得该变量的⼀个失效值。更糟糕的是，失效值可能不会同时出现：⼀个线程可能获得某个变量的最新值，⽽获得另⼀个变量的失效值。

通常，当⾷物过期（即失效）时，还是可以⾷⽤的，只不过味道差了⼀些。但失效的数据可能导致更危险的情况。虽然在Web应⽤程序中失效的命中计数器可能不会导致太糟糕的情况 [2]，但在其他情况中，失效值可能会导致⼀些严重的安全问题或者活跃性问题。在

NoVisibility中，失效数据可能导致输出错误的值，或者使程序⽆法结束。如果对象的引⽤（例如链表中的指针）失效，那么情况会更复杂。失效数据还可能导致⼀些令⼈困惑的故障，例如意料之外的异常、被破坏的数据结构、不精确的计算以及⽆限循环等。

程序清单3-2中的Mutablelnteger不是线程安全的，因为get和set都是在没有同步的情况下访问value的。与其他问题相⽐，失效值问题更容易出现：如果某个线程调⽤了set，那么另⼀个正在调⽤get的线程可能会看到更新后的value值，也可能看不到。

**程序清单3-2 ⾮线程安全的可变整数类**

@NotThreadSafe

public class MutableInteger{

private int value；

public int get（）{return value；}

public void set（int value）{this.value=value；}

}

在程序清单3-3的SynchronizedInteger中，通过对get和set等⽅法进⾏同步，可以使MutableInteger成为⼀个线程安全的类。仅对set⽅法进⾏同步是不够的，调⽤get的线程仍然会看⻅失效值。

**程序清单3-3 线程安全的可变整数类**

```java
@ThreadSafe   
public class SynchronizedInteger{ @GuardedBy("this") private int value; public synchronized int get () {return value;} public synchronized void set (int value) {this.value=value;} } 
```
