# 5.3.2 串⾏线程封闭

在java.util.concurrent中实现的各种阻塞队列都包含了⾜够的内部同步机制，从⽽安全地将对象从⽣产者线程发布到消费者线程。

对于可变对象，⽣产者-消费者这种设计与阻塞队列⼀起，促进了串⾏线程封闭，从⽽将对象所有权从⽣产者交付给消费者。线程封闭对象只能由单个线程拥有，但可以通过安全地发布该对象来“转移”所有权。在转移所有权后，也只有另⼀个线程能获得这个对象的访问权限，并且发布对象的线程不会再访问它。这种安全的发布确保了对象状态对于新的所有者来说是可⻅的，并且由于最初的所有者不会再访问它，因此对象将被封闭在新的线程中。新的所有者线程可以对该对象做任意修改，因为它具有独占的访问权。

对象池利⽤了串⾏线程封闭，将对象“借给”⼀个请求线程。只要对象池包含⾜够的内部同步来安全地发布池中的对象，并且只要客户代码本⾝不会发布池中的对象，或者在将对象返回给对象池后就不再使⽤它，那么就可以安全地在线程之间传递所有权。

我们也可以使⽤其他发布机制来传递可变对象的所有权，但必须确保只有⼀个线程能接受被转移的对象。阻塞队列简化了这项⼯作。除此之外，还可以通过ConcurrentMap的原⼦⽅法remove或者AtomicReference的原⼦⽅法compareAndSet来完成这项⼯作。
