# 4.3.4 发布底层的状态变量

当把线程安全性委托给某个对象的底层状态变量时，在什么条件下才可以发布这些变量从⽽使其他类能修改它们？答案仍然取决于在类中对这些变量施加了哪些不变性条件。虽然Counter中的value域可以为任意整数值，但Counter施加的约束条件是只能取正整数，此外递增操作同样约束了下⼀个状态的有效取值范围。如果将value声明为⼀个公有域，那么客户代码可以将它修改为⼀个⽆效值，因此发布value会导致这个类出错。另⼀⽅⾯，如果某个变量表⽰的是当前温度或者最近登录⽤户的ID，那么即使另⼀个类在某个时刻修改了这个值，也不会破坏任何不变性条件，因此发布这个变量也是可以接受的。（这或许不是个好主意，因为发布可变的变量将对下⼀步的开发和派⽣⼦类带来限制，但不会破坏类的线程安全性。）

如果⼀个状态变量是线程安全的，并且没有任何不变性条件来约束它的值，在变量的操作上也不存在任何不允许的状态转换，那么就可以安全地发布这个变量。

例如，发布VisualComponent中的mouseListeners或keyListeners等变量就是安全的。由于VisualComponent并没有在其监听器链表的合法状态上施加任何约束，因此这些域可以声明为公有域或者发布，⽽不会破坏线程安全性。
