# 4.1.2 依赖状态的操作

类的不变性条件与后验条件约束了在对象上有哪些状态和状态转换是有效的。在某些对象的⽅法中还包含⼀些基于状态的先验条件（Precondition）。例如，不能从空队列中移除⼀个元素，在删除元素前，队列必须处于“⾮空的”状态。如果在某个操作中包含有基于状态的先验条件，那么这个操作就称为依赖状态的操作。

在单线程程序中，如果某个操作⽆法满⾜先验条件，那么就只能失败。但在并发程序中，先验条件可能会由于其他线程执⾏的操作⽽变成真。在并发程序中要⼀直等到先验条件为真，然后再执⾏该操作。

在Java中，等待某个条件为真的各种内置机制（包括等待和通知等机制）都与内置加锁机制紧密关联，要想正确地使⽤它们并不容易。要想实现某个等待先验条件为真时才执⾏的操作，⼀种更简单的⽅法是通过现有库中的类（例如阻塞队列[Blocking Queue]或信号量

[Semaphore]）来实现依赖状态的⾏为。第5章将介绍⼀些阻塞类，例如BlockingQueue、Semaphore以及其他的同步⼯具类。第14章将介绍如何使⽤在平台与类库中提供的各种底层机制来创建依赖状态的类。
