# 4.2.1 Java监视器模式

从线程封闭原则及其逻辑推论可以得出Java监视器模式 [1]。遵循Java监视器模式的对象会把对象的所有可变状态都封装起来，并由对象⾃⼰的内置锁来保护。

在程序清单4-1的Counter中给出了这种模式的⼀个典型⽰例。在Counter中封装了⼀个状态变量value，对该变量的所有访问都需要通过

Counter的⽅法来执⾏，并且这些⽅法都是同步的。

在许多类中都使⽤了Java监视器模式，例如Vector和Hashtable。在某些情况下，程序需要⼀种更复杂的同步策略。第11章将介绍如何通过细粒度的加锁策略来提⾼可伸缩性。Java监视器模式的主要优势就在于它的简单性。

Java监视器模式仅仅是⼀种编写代码的约定，对于任何⼀种锁对象，只要⾃始⾄终都使⽤该锁对象，都可以⽤来保护对象的状态。程序清单4-3给出了如何使⽤私有锁来保护状态。
