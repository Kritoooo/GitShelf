# 2.2.3 复合操作

LazyInitRace和UnsafeCountingFactorizer都包含⼀组需要以原⼦⽅式执⾏（或者说不可分割）的操作。要避免竞态条件问题，就必须在某个线程修改该变量时，通过某种⽅式防⽌其他线程使⽤这个变量，从⽽确保其他线程只能在修改操作完成之前或之后读取和修改状态，⽽不是在修改状态的过程中。

假定有两个操作A和B，如果从执⾏A的线程来看，当另⼀个线程执⾏B时，要么将B全部执⾏完，要么完全不执⾏B，那么A和B对彼此来说是原⼦的。原⼦操作是指，对于访问同⼀个状态的所有操作（包括该操作本⾝）来说，这个操作是⼀个以原⼦⽅式执⾏的操作。

如果UnsafeSequence中的递增操作是原⼦操作，那么图1-1中的竞态条件就不会发⽣，并且递增操作在每次执⾏时都会把计数器增加1。为了确保线程安全性，“先检查后执⾏”（例如延迟初始化）和“读取-修改-写⼊”（例如递增运算）等操作必须是原⼦的。我们将“先检查后执⾏”以及“读取-修改-写⼊”等操作统称为复合操作：包含了⼀组必须以原⼦⽅式执⾏的操作以确保线程安全性。在2.3节中，我们将介绍加锁机制，这是Java中⽤于确保原⼦性的内置机制。就⽬前⽽⾔，我们先采⽤另⼀种⽅式来修复这个问题，即使⽤⼀个现有的线程安全类，如程序清单2-4中的CountingFactorizer所⽰。
