# 1.2.1 发挥多处理器的强⼤能⼒

过去，多处理器系统是⾮常昂贵和稀少的，通常只有在⼤型数据中⼼和科学计算设备中才会使⽤多处理器系统。但现在，多处理器系统正⽇益普及，并且价格也在不断地降低，即使在低端服务器和中端桌⾯系统中，通常也会采⽤多个处理器。这种趋势还将进⼀步加快，因为通过提⾼时钟频率来提升性能已变得越来越困难，处理器⽣产⼚商都开始转⽽在单个芯⽚上放置多个处理器核。所有的主流芯⽚制造商都开始了这种转变，⽽我们也已经看到了在⼀些机器上出现了更多的处理器。

由于基本的调度单位是线程，因此如果在程序中只有⼀个线程，那么最多同时只能在⼀个处理器上运⾏。在双处理器系统上，单线程的程序只能使⽤⼀半的CPU资源，⽽在拥有100个处理器的系统上，将有$9 9 \%$ 的资源⽆法使⽤。另⼀⽅⾯，多线程程序可以同时在多个处理器上执⾏。如果设计正确，多线程程序可以通过提⾼处理器资源的利⽤率来提升系统吞吐率。

使⽤多个线程还有助于在单处理器系统上获得更⾼的吞吐率。如果程序是单线程的，那么当程序等待某个同步I/O操作完成时，处理器将处于空闲状态。⽽在多线程程序中，如果⼀个线程在等待I/O操作完成，另⼀个线程可以继续运⾏，使程序能够在I/O阻塞期间继续运⾏。（这就好⽐在等待⽔烧开的同时看报纸，⽽不是等到⽔烧开之后再开始看报纸）。
