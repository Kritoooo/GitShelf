# 5.3.3 双端队列与⼯作密取

Java 6增加了两种容器类型，Deque（发⾳为“deck”）和BlockingDeque，它们分别对Queue和BlockingQueue进⾏了扩展。Deque是⼀个双端队列，实现了在队列头和队列尾的⾼效插⼊和移除。具体实现包括ArrayDeque和LinkedBlockingDeque。

正如阻塞队列适⽤于⽣产者-消费者模式，双端队列同样适⽤于另⼀种相关模式，即⼯作密取（Work Stealing）。在⽣产者-消费者设计中，所有消费者有⼀个共享的⼯作队列，⽽在⼯作密取设计中，每个消费者都有各⾃的双端队列。如果⼀个消费者完成了⾃⼰双端队列中的全部⼯作，那么它可以从其他消费者双端队列末尾秘密地获取⼯作。密取⼯作模式⽐传统的⽣产者-消费者模式具有更⾼的可伸缩性，这是因为⼯作者线程不会在单个共享的任务队列上发⽣竞争。在⼤多数时候，它们都只是访问⾃⼰的双端队列，从⽽极⼤地减少了竞争。当⼯作者线程需要访问另⼀个队列时，它会从队列的尾部⽽不是从头部获取⼯作，因此进⼀步降低了队列上的竞争程度。

⼯作密取⾮常适⽤于既是消费者也是⽣产者问题——当执⾏某个⼯作时可能导致出现更多的⼯作。例如，在⽹⻚爬⾍程序中处理⼀个⻚⾯时，通常会发现有更多的⻚⾯需要处理。类似的还有许多搜索图的算法，例如在垃圾回收阶段对堆进⾏标记，都可以通过⼯作密取机制来实现⾼效并⾏。当⼀个⼯作线程找到新的任务单元时，它会将其放到⾃⼰队列的末尾（或者在⼯作共享设计模式中，放⼊其他⼯作者线程的队列中）。当双端队列为空时，它会在另⼀个线程的队列队尾查找新的任务，从⽽确保每个线程都保持忙碌状态。
