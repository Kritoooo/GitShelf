# 5.2 并发容器

Java 5. 0提供了多种并发容器类来改进同步容器的性能。同步容器将所有对容器状态的访问都串⾏化，以实现它们的线程安全性。这种⽅法的代价是严重降低并发性，当多个线程竞争容器的锁时，吞吐量将严重减低。

另⼀⽅⾯，并发容器是针对多个线程并发访问设计的。在Java 5.0中增加了Concurrent-HashMap，⽤来替代同步且基于散列的Map，以及CopyOnWriteArrayList，⽤于在遍历操作为主要操作的情况下代替同步的List。在新的ConcurrentMap接⼝中增加了对⼀些常⻅复合操作的⽀持，例如“若没有则添加”、替换以及有条件删除等。

通过并发容器来代替同步容器，可以极⼤地提⾼伸缩性并降低⻛险。

Java 5. 0增加了两种新的容器类型：Queue和BlockingQueue。Queue⽤来临时保存⼀组等待处理的元素。它提供了⼏种实现，包括：ConcurrentLinkedQueue，这是⼀个传统的先进先出队列，以及PriorityQueue，这是⼀个（⾮并发的）优先队列。Queue上的操作不会阻塞，如果队列为空，那么获取元素的操作将返回空值。虽然可以⽤List来模拟Queue的⾏为——事实上，正是通过LinkedList来实现Queue的，但还需要⼀个Queue的类，因为它能去掉List的随机访问需求，从⽽实现更⾼效的并发。

BlockingQueue扩展了Queue，增加了可阻塞的插⼊和获取等操作。如果队列为空，那么获取元素的操作将⼀直阻塞，直到队列中出现⼀个可⽤的元素。如果队列已满（对于有界队列来说），那么插⼊元素的操作将⼀直阻塞，直到队列中出现可⽤的空间。在“⽣产者-消费者”这种设计模式中，阻塞队列是⾮常有⽤的，5.3节将会详细介绍。

正如ConcurrentHashMap⽤于代替基于散列的同步Map, Java 6也引⼊了Concurrent-SkipListMap和ConcurrentSkipListSet，分别作为同步的SortedMap和SortedSet的并发替代品（例如⽤synchronizedMap包装的TreeMap或TreeSet）。
