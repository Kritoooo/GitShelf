# 5.2.2 额外的原⼦Map操作

由于ConcurrentHashMap不能被加锁来执⾏独占访问，因此我们⽆法使⽤客户端加锁来创建新的原⼦操作，例如4.4.1节中对Vector增加原⼦操作“若没有则添加”。但是，⼀些常⻅的复合操作，例如“若没有则添加”、“若相等则移除（Remove-If-Equal）”和“若相等则替换（Replace-If-Equal）”等，都已经实现为原⼦操作并且在ConcurrentMap的接⼝中声明，如程序清单5-7所⽰。如果你需要在现有的同步Map中添加这样的功能，那么很可能就意味着应该考虑使⽤ConcurrentMap了。

**程序清单5-7 ConcurrentMap接⼝**

```txt
public interface ConcurrentMap<K, V> extends Map<K, V> {  
//仅当K没有相应的映射值时才插入  
V putIfAbsent (K key, V value);  
//仅当K被映射到V时才移除  
boolean remove (K key, V value);  
//仅当K被映射到oldValue时才替换为newValue  
boolean replace (K key, V oldValue, V newVal);  
//仅当K被映射到某个值时才替换为 newVal  
V replace (K key, V newVal);  
} 
```
