# 3.2 发布与逸出

“发布（Publish）”⼀个对象的意思是指，使对象能够在当前作⽤域之外的代码中使⽤。例如，将⼀个指向该对象的引⽤保存到其他代码可以访问的地⽅，或者在某⼀个⾮私有的⽅法中返回该引⽤，或者将引⽤传递到其他类的⽅法中。在许多情况中，我们要确保对象及其内部状态不被发布。⽽在某些情况下，我们⼜需要发布某个对象，但如果在发布时要确保线程安全性，则可能需要同步。发布内部状态可能会破坏封装性，并使得程序难以维持不变性条件。例如，如果在对象构造完成之前就发布该对象，就会破坏线程安全性。当某个不应该发布的对象被发布时，这种情况就被称为逸出（Escape）。3.5节介绍了如何安全发布对象的⼀些⽅法。现在，我们⾸先来看看⼀个对象是如何逸出的。

发布对象的最简单⽅法是将对象的引⽤保存到⼀个公有的静态变量中，以便任何类和线程都能看⻅该对象，如程序清单3-5所⽰。在initialize⽅法中实例化⼀个新的HashSet对象，并将对象的引⽤保存到knownSecrets中以发布该对象。

**程序清单3-5 发布⼀个对象**


当发布某个对象时，可能会间接地发布其他对象。如果将⼀个Secret对象添加到集合knownSecrets中，那么同样会发布这个对象，因为任何代码都可以遍历这个集合，并获得对这个新Secret对象的引⽤。同样，如果从⾮私有⽅法中返回⼀个引⽤，那么同样会发布返回的对象。程序清单3-6中的UnsafeStates发布了本应为私有的状态数组。

**安全的对象构造过程**

在ThisEscape中给出了逸出的⼀个特殊⽰例，即this引⽤在构造函数中逸出。当内部的EventListener实例发布时，在外部封装的ThisEscape实例也逸出了。当且仅当对象的构造函数返回时，对象才处于可预测的和⼀致的状态。因此，当从对象的构造函数中发布对象时，只是发布了⼀个尚未构造完成的对象。即使发布对象的语句位于构造函数的最后⼀⾏也是如此。如果this引⽤在构造过程中逸出，那么这种对象就被认为是不正确构造[2]。

不要在构造过程中使this引⽤逸出。

在构造过程中使this引⽤逸出的⼀个常⻅错误是，在构造函数中启动⼀个线程。当对象在其构造函数中创建⼀个线程时，⽆论是显式创建（通过将它传给构造函数）还是隐式创建（由于Thread或Runnable是该对象的⼀个内部类），this引⽤都会被新创建的线程共享。在对象尚未完全构造之前，新的线程就可以看⻅它。在构造函数中创建线程并没有错误，但最好不要⽴即启动它，⽽是通过⼀个start或initialize⽅法来启动（请参⻅第7章了解更多关于服务⽣命周期的内容）。在构造函数中调⽤⼀个可改写的实例⽅法时（既不是私有⽅法，也不是终结⽅法），同样会导致this引⽤在构造过程中逸出。

如果想在构造函数中注册⼀个事件监听器或启动线程，那么可以使⽤⼀个私有的构造函数和⼀个公共的⼯⼚⽅法（Factory Method），从⽽避免不正确的构造过程，如程序清单3-8中SafeListener所⽰。
