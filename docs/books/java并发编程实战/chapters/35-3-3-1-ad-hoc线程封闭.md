# 3.3.1 Ad-hoc线程封闭

Ad-hoc线程封闭是指，维护线程封闭性的职责完全由程序实现来承担。Ad-hoc线程封闭是⾮常脆弱的，因为没有任何⼀种语⾔特性，例如可⻅性修饰符或局部变量，能将对象封闭到⽬标线程上。事实上，对线程封闭对象（例如，GUI应⽤程序中的可视化组件或数据模型等）的引⽤通常保存在公有变量中。

当决定使⽤线程封闭技术时，通常是因为要将某个特定的⼦系统实现为⼀个单线程⼦系统。在某些情况下，单线程⼦系统提供的简便性要胜过Ad-hoc线程封闭技术的脆弱性。 [2]

在volatile变量上存在⼀种特殊的线程封闭。只要你能确保只有单个线程对共享的volatile变量执⾏写⼊操作，那么就可以安全地在这些共享的volatile变量上执⾏“读取-修改-写⼊”的操作。在这种情况下，相当于将修改操作封闭在单个线程中以防⽌发⽣竞态条件，并且volatile变量的可⻅性保证还确保了其他线程能看到最新的值。

由于Ad-hoc线程封闭技术的脆弱性，因此在程序中尽量少⽤它，在可能的情况下，应该使⽤更强的线程封闭技术（例如，栈封闭或ThreadLocal类）。

[1] 应⽤程序服务器提供的连接池是线程安全的。连接池通常会由多个线程同时访问，因此⾮线程安全的连接池是毫⽆意义的。  
[2] 使⽤单线程⼦系统的另⼀个原因是为了避免死锁，这也是⼤多数GUI框架都是单线程的原因。第9章将进⼀步介绍单线程⼦系统。
