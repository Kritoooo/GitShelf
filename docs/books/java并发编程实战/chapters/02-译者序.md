# 译者序

并发编程是Java语⾔的重要特性之⼀，在Java平台上提供了许多基本的并发功能来辅助开发多线程应⽤程序。然⽽，这些相对底层的并发功能与上层应⽤程序的并发语义之间并不存在⼀种简单⽽直观的映射关系。因此，如何在Java并发应⽤程序中正确且⾼效地使⽤这些功能就成了Java开发⼈员的关注重点。

本书正是为了解决这个问题⽽写的。书中采⽤循序渐进的讲解⽅式，从并发编程的基本理论⼊⼿，逐步介绍了在设计Java并发程序时各种重要的设计原则、设计模式以及思维模式，同时辅以丰富的⽰例代码作为对照和补充，使得开发⼈员能够更快地领悟Java并发编程的要领，围绕着Java平台的基础并发功能快速地构建⼤规模的并发应⽤程序。

全书内容由浅⼊深，共分为四个部分。第⼀部分介绍了Java并发编程的基础理论，包括线程安全性与状态对象的基础知识，如何构造线程安全的类并将多个⼩型的线程安全类构建成更⼤型的线程安全类，以及Java平台库中的⼀些基础并发模块；第⼆部分介绍了并发应⽤程序的构造理论，包括应⽤程序中并⾏语义的分解及其与逻辑任务的映射，任务的取消与关闭等⾏为的实现，以及Java线程池中的⼀些⾼级功能，此外还介绍了如何提⾼GUI应⽤程序的响应性；第三部分介绍了并发编程的性能调优，包括如何避免活跃性问题，如何提⾼并发代码的性能和可伸缩性以获得理想的性能，以及在测试并发代码正确性和性能时的⼀些实⽤技术；第四部分介绍了Java并发编程中的⼀些⾼级主题，包括显式锁、原⼦变量、⾮阻塞算法以及如何开发⾃定义的同步⼯具类等。

本书的特点在于注重阐述并发技术背后的理论知识，对于每种技术的介绍不仅使读者能做到“知其然”，更能做到“知其所以然”。对于希望

深⼊研究和探索Java并发编程的读者来说，本书是⾮常合适的。

参与本书翻译⼯作的还有李杨、吴汉平、徐光景、童胜汉、陈军、胡凯、刘红、张玮、陈红、李斌、李勇涛、王海涛、周云波、彭敏才、张世锋、朱介秋、宗敬、李静、叶锦、⾼波、熊莉、程凤、陈娟、胡世娟、董敏、谢路阳、冯卓、李志勇、胡欢、王进等。由于译者的时间和⽔平有限，翻译中的疏漏和错误在所难免，还望读者和同⾏不吝指正。

**前⾔**

在写作本书时，对于中端桌⾯系统来说，多核处理器正变得越来越便宜。⽆独有偶，许多开发团队也注意到，在他们的项⽬中出现了越来越多与线程有关的错误报告。在NetBeans开发者⽹站上的最近⼀次公告中，⼀位核⼼维护⼈员注意到，为了修复与线程相关的问题，在某个类中竟然打了14次补丁。Dion Almaer，这位TheServerSide⽹站的前编辑，最近（在经过⼀番痛苦的调试过程并最终发现了⼀个与线程有关的错误之后）在其博客上写道，在⼤多数Java程序中充满了各种并发错误，使得程序只有在“偶然的情况下”才能正常⼯作。

确实，在开发、测试以及调试多线程程序时存在着巨⼤的困难，因为并发性错误通常并不会以某种确定的⽅式显现出来。当这些错误出现时，通常是在最糟糕的时刻，例如在正式产品中，或者在⾼负载的情况下。

当开发Java并发程序时，所要⾯对的挑战之⼀就是：平台提供的各种并发功能与开发⼈员在程序中需要的并发语义并不匹配。在Java语⾔中提供了⼀些底层机制，例如同步和条件等待，但在使⽤这些机制来实现应⽤级的协议与策略时必须始终保持⼀致。如果没有这些策略，那么在编写程序时，虽然程序看似能顺利地编译和运⾏，但却总会出现各种奇怪的问题。许多介绍并发的其他书籍更侧重于介绍⼀些底层机制和API，⽽在设计级的策略和模式上叙述的不多。

Java 5. 0在Java并发应⽤程序的开发⽅⾯进展巨⼤，它不仅提供了⼀些新的⾼层组件，还补充了⼀些底层机制，从⽽使得⽆论是新⼿级开发⼈员还是专家级开发⼈员都能够更容易地构建并发应⽤程序。本书的作者都是JCP专家组的主要成员，也正是该专家组编写了这些新功能。本书不仅描述了这些新功能的⾏为和特性，还介绍了它们的底层设计模式和促使它们被添加到平台库中的应⽤场景。

我们的⽬标是向读者介绍⼀些设计规则和思维模式，从⽽使读者能够更容易也更乐意去构建正确的以及⾼性能的Java并发类和应⽤程序。

我们希望你能享受本书的阅读过程。

**如何使⽤本书**

为了解决在Java底层机制与设计级策略之间的不匹配问题，我们给出了⼀组简化的并发程序编写规则。专家看到这些规则会说：“嗯，这并不是完整的规则集。即使类C违背了规则R，它仍然是线程安全的。”虽然在违背⼀些规则的情况下仍有可能编写出正确的并发程序，但这需要对Java内存模型的底层细节有着深⼊的理解，⽽我们希望开发⼈员⽆须掌握这些细节就能编写出正确的并发程序。只要始终遵循这组简单的规则，就能编写出正确的并且可维护的并发程序。

我们假设读者对Java的基本并发机制已经有了⼀定程度的了解。本书并⾮是对并发的⼊门介绍—要了解这⽅⾯的内容，请参考其他书籍中有关线程的内容，例如《The Java Programming Language》（Arnold等，2005）。此外，本书也不是介绍并发的百科全书—要了解这⽅⾯的内容，请参考《Concurrent Programming in Java》（Lea，2000）。事实上，本书提供了各种实⽤的设计规则，⽤于帮助开发⼈员创建安全的和⾼性能的并发类。在本书中相应的地⽅引⽤了以下书籍中的相关章节：《The Java Programming Language》、《ConcurrentProgramming in Java》、《The Java Language Specification》（Gosling等，2005）以及《Effective Java》（Bloch，2001），并分别使⽤[JPLn.m]、[CPJ n.m]、[JLS n.m]和[EJ Item n]来表⽰它们。

在进⾏简要的介绍（第1章）之后，本书共分为四个部分：

基础知识。第⼀部分（第2章〜第5章）重点介绍了并发性和线程安全性的基本概念，以及如何使⽤类库提供的基本并发构建块来构建线程安全类。在第⼀部分给出了⼀个清单，其中总结了这⼀部分中介绍的最重要的规则。

活跃性、性能与测试。第三部分（第10章〜第12章）介绍了如何确保并发程序执⾏预期的任务，以及如何获得理想的性能。第10章介绍了如何避免⼀些使程序⽆法执⾏下去的活跃性故障。第11章介绍了如何提⾼并发代码的性能和可伸缩性。第12章介绍了在测试并发代码的正确性和性能时可以采⽤的⼀些技术。
