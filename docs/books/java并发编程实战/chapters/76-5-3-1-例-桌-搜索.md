⽣产者-消费者模式提供了⼀种适合线程的⽅法将桌⾯搜索问题分解为更简单的组件。将⽂件遍历与建⽴索引等功能分解为独⽴的操作，⽐将所有功能都放到⼀个操作中实现有着更⾼的代码可读性和可重⽤性：每个操作只需完成⼀个任务，并且阻塞队列将负责所有的控制流，因此每个功能的代码都更加简单和清晰。

⽣产者-消费者模式同样能带来许多性能优势。⽣产者和消费者可以并发地执⾏。如果⼀个是I/O密集型，另⼀个是CPU密集型，那么并发执⾏的吞吐率要⾼于串⾏执⾏的吞吐率。如果⽣产者和消费者的并⾏度不同，那么将它们紧密耦合在⼀起会把整体并⾏度降低为⼆者中更⼩的并⾏度。

在程序清单5-9中启动了多个爬⾍程序和索引建⽴程序，每个程序都在各⾃的线程中运⾏。前⾯曾讲，消费者线程永远不会退出，因⽽程序⽆法终⽌，第7章将介绍多种技术来解决这个问题。虽然这个⽰例使⽤了显式管理的线程，但许多⽣产者-消费者设计也可以通过Executor任务执⾏框架来实现，其本⾝也使⽤了⽣产者-消费者模式。
