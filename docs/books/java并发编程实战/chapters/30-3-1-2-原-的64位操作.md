# 3.1.2 ⾮原⼦的64位操作

当线程在没有同步的情况下读取变量时，可能会得到⼀个失效值，但⾄少这个值是由之前某个线程设置的值，⽽不是⼀个随机值。这种安全性保证也被称为最低安全性（out-of-thin-air safety）。

最低安全性适⽤于绝⼤多数变量，但是存在⼀个例外：⾮volatile类型的64位数值变量（double和long，请参⻅3.1.4节）。Java内存模型要求，变量的读取操作和写⼊操作都必须是原⼦操作，但对于⾮volatile类型的long和double变量，JVM允许将64位的读操作或写操作分解为两个32位的操作。当读取⼀个⾮volatile类型的long变量时，如果对该变量的读操作和写操作在不同的线程中执⾏，那么很可能会读取到某个值的⾼32位和另⼀个值的低32位[1]。因此，即使不考虑失效数据问题，在多线程程序中使⽤共享且可变的long和double等类型的变量也是不安全的，除⾮⽤关键字volatile来声明它们，或者⽤锁保护起来。

[1] 在编写Java虚拟机规范时，许多主流处理器架构还不能有效地提供64位数值的原⼦操作。
