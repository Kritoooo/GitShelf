# 4.4 在现有的线程安全类中添加功能

Java类库包含许多有⽤的“基础模块”类。通常，我们应该优先选择重⽤这些现有的类⽽不是创建新的类：重⽤能降低开发⼯作量、开发⻛险（因为现有的类都已经通过测试）以及维护成本。有时候，某个现有的线程安全类能⽀持我们需要的所有操作，但更多时候，现有的类只能⽀持⼤部分的操作，此时就需要在不破坏线程安全性的情况下添加⼀个新的操作。

例如，假设需要⼀个线程安全的链表，它需要提供⼀个原⼦的“若没有则添加（Put-If-Absent）”的操作。同步的List类已经实现了⼤部分的功能，我们可以根据它提供的contains⽅法和add⽅法来构造⼀个“若没有则添加”的操作。

“若没有则添加”的概念很简单，在向容器中添加元素前，⾸先检查该元素是否已经存在，如果存在就不再添加。（回想“先检查再执⾏”的注意事项。）由于这个类必须是线程安全的，因此就隐含地增加了另⼀个需求，即“若没有则添加”这个操作必须是原⼦操作。这意味着，如果在链表中没有包含对象X，那么在执⾏两次“若没有则添加”X后，在容器中只能包含⼀个X对象。然⽽，如果“若没有则添加”操作不是原⼦操作，那么在某些执⾏情况下，有两个线程都将看到X不在容器中，并且都执⾏了添加X的操作，从⽽使容器中包含两个相同的X对象。

要添加⼀个新的原⼦操作，最安全的⽅法是修改原始的类，但这通常⽆法做到，因为你可能⽆法访问或修改类的源代码。要想修改原始的类，就需要理解代码中的同步策略，这样增加的功能才能与原有的设计保持⼀致。如果直接将新⽅法添加到类中，那么意味着实现同步策略的所有代码仍然处于⼀个源代码⽂件中，从⽽更容易理解与维护。

另⼀种⽅法是扩展这个类，假定在设计这个类时考虑了可扩展性。程序清单4-13中的BetterVector对Vector进⾏了扩展，并添加了⼀个新⽅法putIfAbsent。扩展Vector很简单，但并⾮所有的类都像Vector那样将状态向⼦类公开，因此也就不适合采⽤这种⽅法。
