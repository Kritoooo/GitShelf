# 5.2.3 CopyOnWriteArrayList

CopyOnWriteArrayList⽤于替代同步List，在某些情况下它提供了更好的并发性能，并且在迭代期间不需要对容器进⾏加锁或复制。（类似地，CopyOnWriteArraySet的作⽤是替代同步Set。）

“写⼊时复制（Copy-On-Write）”容器的线程安全性在于，只要正确地发布⼀个事实不可变的对象，那么在访问该对象时就不再需要进⼀步的同步。在每次修改时，都会创建并重新发布⼀个新的容器副本，从⽽实现可变性。“写⼊时复制”容器的迭代器保留⼀个指向底层基础数组的引⽤，这个数组当前位于迭代器的起始位置，由于它不会被修改，因此在对其进⾏同步时只需确保数组内容的可⻅性。因此，多个线程可以同时对这个容器进⾏迭代，⽽不会彼此⼲扰或者与修改容器的线程相互⼲扰。“写⼊时复制”容器返回的迭代器不会抛出ConcurrentModificationException，并且返回的元素与迭代器创建时的元素完全⼀致，⽽不必考虑之后修改操作所带来的影响。

显然，每当修改容器时都会复制底层数组，这需要⼀定的开销，特别是当容器的规模较⼤时。仅当迭代操作远远多于修改操作时，才应该使⽤“写⼊时复制”容器。这个准则很好地描述了许多事件通知系统：在分发通知时需要迭代已注册监听器链表，并调⽤每⼀个监听器，在⼤多数情况下，注册和注销事件监听器的操作远少于接收事件通知的操作。（关于“写⼊时复制”的更多信息请参⻅[CPJ 2.4.4]。）
