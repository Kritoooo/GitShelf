# 3.5.4 事实不可变对象

如果对象在发布后不会被修改，那么对于其他在没有额外同步的情况下安全地访问这些对象的线程来说，安全发布是⾜够的。所有的安全发布机制都能确保，当对象的引⽤对所有访问该对象的线程可⻅时，对象发布时的状态对于所有线程也将是可⻅的，并且如果对象状态不会再改变，那么就⾜以确保任何访问都是安全的。

如果对象从技术上来看是可变的，但其状态在发布后不会再改变，那么把这种对象称为“事实不可变对象（Effectively ImmutableObject）”。这些对象不需要满⾜3.4节中提出的不可变性的严格定义。在这些对象发布后，程序只需将它们视为不可变对象即可。通过使⽤事实不可变对象，不仅可以简化开发过程，⽽且还能由于减少了同步⽽提⾼性能。

在没有额外的同步的情况下，任何线程都可以安全地使⽤被安全发布的事实不可变对象。

例如，Date本⾝是可变的 [1]，但如果将它作为不可变对象来使⽤，那么在多个线程之间共享Date对象时，就可以省去对锁的使⽤。假设需要维护⼀个Map对象，其中保存了每位⽤户的最近登录时间：

```java
public Map<String, Date>lastLogin= 
```

```txt
Collections.synchronizedMap(new HashMap<String, Date>) ; 
```

如果Date对象的值在被放⼊Map后就不会改变，那么synchronizedMap中的同步机制就⾜以使Date值被安全地发布，并且在访问这些Date值时不需要额外的同步。

[1] 这或许是类库设计中的⼀个错误。
