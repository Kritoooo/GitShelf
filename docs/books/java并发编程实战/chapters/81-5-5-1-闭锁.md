# 5.5.1 闭锁

闭锁是⼀种同步⼯具类，可以延迟线程的进度直到其到达终⽌状态[CPJ 3.4.2]。闭锁的作⽤相当于⼀扇门：在闭锁到达结束状态之前，这扇门⼀直是关闭的，并且没有任何线程能通过，当到达结束状态时，这扇门会打开并允许所有的线程通过。当闭锁到达结束状态后，将不会再改变状态，因此这扇门将永远保持打开状态。闭锁可以⽤来确保某些活动直到其他活动都完成后才继续执⾏，例如：

·确保某个计算在其需要的所有资源都被初始化之后才继续执⾏。⼆元闭锁（包括两个状态）可以⽤来表⽰“资源R已经被初始化”，⽽所有需要R的操作都必须先在这个闭锁上等待。

·确保某个服务在其依赖的所有其他服务都已经启动之后才启动。每个服务都有⼀个相关的⼆元闭锁。当启动服务S时，将⾸先在S依赖的其他服务的闭锁上等待，在所有依赖的服务都启动后会释放闭锁S，这样其他依赖S的服务才能继续执⾏。

·等待直到某个操作的所有参与者（例如，在多玩家游戏中的所有玩家）都就绪再继续执⾏。在这种情况中，当所有玩家都准备就绪时，闭锁将到达结束状态。

CountDownLatch是⼀种灵活的闭锁实现，可以在上述各种情况中使⽤，它可以使⼀个或多个线程等待⼀组事件发⽣。闭锁状态包括⼀个计数器，该计数器被初始化为⼀个正数，表⽰需要等待的事件数量。countDown⽅法递减计数器，表⽰有⼀个事件已经发⽣了，⽽await⽅法等待计数器达到零，这表⽰所有需要等待的事件都已经发⽣。如果计数器的值⾮零，那么await会⼀直阻塞直到计数器为零，或者等待中的线程中断，或者等待超时。

在程序清单5-11的TestHarness中给出了闭锁的两种常⻅⽤法。

TestHarness创建⼀定数量的线程，利⽤它们并发地执⾏指定的任务。它使⽤两个闭锁，分别表⽰“起始门（Starting Gate）”和“结束门（Ending Gate）”。起始门计数器的初始值为1，⽽结束门计数器的初始值为⼯作线程的数量。每个⼯作线程⾸先要做的值就是在启动门上等待，从⽽确保所有线程都就绪后才开始执⾏。⽽每个线程要做的最后⼀件事情是将调⽤结束门的countDown⽅法减1，这能使主线程⾼效地等待直到所有⼯作线程都执⾏完成，因此可以统计所消耗的时间。

程序清单5-11 在计时测试中使⽤CountDownLatch来启动和停⽌线程

为什么要在TestHarness中使⽤闭锁，⽽不是在线程创建后就⽴即启动？或许，我们希望测试n个线程并发执⾏某个任务时需要的时间。如果在创建线程后⽴即启动它们，那么先启动的线程将“领先”后启动的线程，并且活跃线程数量会随着时间的推移⽽增加或减少，竞争程度也在不断发⽣变化。启动门将使得主线程能够同时释放所有⼯作线程，⽽结束门则使主线程能够等待最后⼀个线程执⾏完成，⽽不是顺序地等待每个线程执⾏完成。
