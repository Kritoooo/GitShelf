通过添加⼀个原⼦操作来扩展类是脆弱的，因为它将类的加锁代码分布到多个类中。然⽽，客户端加锁却更加脆弱，因为它将类C的加锁代码放到与C完全⽆关的其他类中。当在那些并不承诺遵循加锁策略的类上使⽤客户端加锁时，要特别⼩⼼。

客户端加锁机制与扩展类机制有许多共同点，⼆者都是将派⽣类的⾏为与基类的实现耦合在⼀起。正如扩展会破坏实现的封装性[EJ Item14]，客户端加锁同样会破坏同步策略的封装性。
