# 1.3.3 性能问题

与活跃性问题密切相关的是性能问题。活跃性意味着某件正确的事情最终会发⽣，但却不够好，因为我们通常希望正确的事情尽快发⽣。性能问题包括多个⽅⾯，例如服务时间过⻓，响应不灵敏，吞吐率过低，资源消耗过⾼，或者可伸缩性较低等。与安全性和活跃性⼀样，在多线程程序中不仅存在与单线程程序相同的性能问题，⽽且还存在由于使⽤线程⽽引⼊的其他性能问题。

在设计良好的并发应⽤程序中，线程能提升程序的性能，但⽆论如何，线程总会带来某种程度的运⾏时开销。在多线程程序中，当线程调度器临时挂起活跃线程并转⽽运⾏另⼀个线程时，就会频繁地出现上下⽂切换操作（Context Switch），这种操作将带来极⼤的开销：保存和恢复执⾏上下⽂，丢失局部性，并且CPU时间将更多地花在线程调度⽽不是线程运⾏上。当线程共享数据时，必须使⽤同步机制，⽽这些机制往往会抑制某些编译器优化，使内存缓存区中的数据⽆效，以及增加共享内存总线的同步流量。所有这些因素都将带来额外的性能开销，第11章将详细介绍如何分析和减少这些开销。
