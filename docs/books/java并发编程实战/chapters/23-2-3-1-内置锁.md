# 2.3.1 内置锁

Java提供了⼀种内置的锁机制来⽀持原⼦性：同步代码块（Synchronized Block）。（第3章将介绍加锁机制以及其他同步机制的另⼀个重要⽅⾯：可⻅性）同步代码块包括两部分：⼀个作为锁的对象引⽤，⼀个作为由这个锁保护的代码块。以关键字synchronized来修饰的⽅法就是⼀种横跨整个⽅法体的同步代码块，其中该同步代码块的锁就是⽅法调⽤所在的对象。静态的synchronized⽅法以Class对象作为锁。

每个Java对象都可以⽤做⼀个实现同步的锁，这些锁被称为内置锁（Intrinsic Lock）或监视器锁（Monitor Lock）。线程在进⼊同步代码块之前会⾃动获得锁，并且在退出同步代码块时⾃动释放锁，⽽⽆论是通过正常的控制路径退出，还是通过从代码块中抛出异常退出。获得内置锁的唯⼀途径就是进⼊由这个锁保护的同步代码块或⽅法。

Java的内置锁相当于⼀种互斥体（或互斥锁），这意味着最多只有⼀个线程能持有这种锁。当线程A尝试获取⼀个由线程B持有的锁时，线程A必须等待或者阻塞，直到线程B释放这个锁。如果B永远不释放锁，那么A也将永远地等下去。

由于每次只能有⼀个线程执⾏内置锁保护的代码块，因此，由这个锁保护的同步代码块会以原⼦⽅式执⾏，多个线程在执⾏该代码块时也不会相互⼲扰。并发环境中的原⼦性与事务应⽤程序中的原⼦性有着相同的含义—⼀组语句作为⼀个不可分割的单元被执⾏。任何⼀个执⾏同步代码块的线程，都不可能看到有其他线程正在执⾏由同⼀个锁保护的同步代码块。
