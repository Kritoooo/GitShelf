# 3.5 安全发布

到⽬前为⽌，我们重点讨论的是如何确保对象不被发布，例如让对象封闭在线程或另⼀个对象的内部。当然，在某些情况下我们希望在多个线程间共享对象，此时必须确保安全地进⾏共享。然⽽，如果只是像程序清单3-14那样将对象引⽤保存到公有域中，那么还不⾜以安全地发布这个对象。

程序清单3-14 在没有⾜够同步的情况下发布对象（不要这么做）

//不安全的发布

public Holder holder；

public void initialize（）{

holder=new Holder（42）；

}

你可能会奇怪，这个看似没有问题的⽰例何以会运⾏失败。由于存在可⻅性问题，其他线程看到的Holder对象将处于不⼀致的状态，即便在该对象的构造函数中已经正确地构建了不变性条件。这种不正确的发布导致其他线程看到尚未创建完成的对象。
